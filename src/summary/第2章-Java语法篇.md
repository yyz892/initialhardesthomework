## 2.1 字面量

```java
整数: 10, -20, 0
小数: 3.14, -2.5
字符: 'A', '中', '9'        // 单引号，有且仅有一个字符
字符串: "Hello", "中国"       // 双引号，0到多个字符
布尔: true, false
空值: null                  // 不能直接打印
```

**为什么字符和字符串是两种类型？**
- 字符 char 是基本类型，存的是单个字符的Unicode编码（数字）
- 字符串 String 是引用类型，是一个"字符序列"对象
- 底层实现方式完全不同

## 2.2 变量

```java
// 格式：数据类型 变量名 = 初始值;
int age = 18;
double salary = 8000.5;
String name = "张三";

// 先声明再赋值（局部变量必须初始化才能用）
int score;
score = 95;
// System.out.println(x);  // ❌ 没赋值就用，编译报错
```

**为什么变量必须声明类型？**
- Java是**强类型语言**，编译器需要知道每个变量的类型才能检查错误
- 类型写明了，写代码时传错类型编译器马上报错，不用等到运行时才发现
- 这与Python/JavaScript的"弱类型"完全不同

**为什么有作用域限制？**
- 变量在 `{}` 内有效，出了大括号就销毁
- 好处：用完了自动释放内存，两个同名变量在不同 `{}` 里不冲突

## 2.3 8种基本数据类型

### 为什么设计这么多整数类型？

```java
byte  b = 10;    // 1字节，存小数字，省内存
short s = 100;   // 2字节
int   i = 1000;  // 4字节，默认选择
long  l = 100L;  // 8字节，存大数字要加L
```

| 类型 | 字节 | 什么时候用 |
|------|------|-----------|
| byte | 1 | 文件字节流、网络传输，省内存 |
| short | 2 | 很少用，int基本够 |
| int | 4 | **整数默认选int**，够用 |
| long | 8 | 时间戳、金额（分）、大数据量 |

**为什么要分大小？**
为了**省内存**。一个游戏有100万个角色，血量用byte存（1字节/人）vs 用int存（4字节/人），差300万字节≈3MB。这点差别在高并发服务里会被放大很多倍。

### 为什么要有浮点类型？

```java
float  f = 3.14F;   // 4字节，精度较低
double d = 3.14;    // 8字节，默认（加不加D都行）

// ⚠️ 浮点数有精度问题：
System.out.println(0.1 + 0.2);  // 0.30000000000000004，不是0.3！
```

**为什么0.1+0.2不等于0.3？**
计算机用二进制存小数，有些十进制小数无法精确表示为二进制（就像1/3无法精确表示为十进制0.333...）。银行业务用 `BigDecimal`，不用float/double。

### 为什么char占2字节？

C语言的char只占1字节，只能存ASCII（英文+符号）。Java的char用Unicode编码占2字节，可以存中文、日文等所有字符。**这是Java对国际化的原生支持**。

## 2.4 标识符

```java
// ✅ 正确
int age;
int studentAge;          // 小驼峰：变量、方法
String StudentName;      // 大驼峰：类名

// ❌ 错误
int 1age;     // 不能数字开头 → 编译器没法区分是数字还是变量名
int my-age;   // 减号是运算符 → 会解析成 my 减 age
int class;    // 关键字有特殊含义
```

**为什么区分大小驼峰？**
行业约定，一眼能看出是类（大驼峰）还是变量/方法（小驼峰），提高可读性。

## 2.5 Scanner 键盘录入

```java
import java.util.Scanner;

Scanner sc = new Scanner(System.in);
int num = sc.nextInt();
double d = sc.nextDouble();
String str = sc.next();      // 遇空格/回车停止
String line = sc.nextLine(); // 读取整行（含空格）
```

**为什么需要Scanner？**
`System.in` 是原始的字节输入流，程序员没法直接用字节跟用户交互。Scanner帮你把字节流解析成int、double、String。

**为什么 main 方法不能直接读取键盘？**
main只是程序的入口，接收的是启动时传的参数（args）。交互式输入需要额外工具，没有直接集成是为了保持核心语言的简洁。

## 2.6 运算符详解

### 算术运算符

```java
10 / 3  → 3    // 整数除法：舍去小数部分
10 % 3  → 1    // 取模（余数）

// 为什么整数除法这样设计？
// Java保持和C/C++一致的行为，C这样设计是因为早期计算机的整数除法电路就这么做
```

### 类型转换

```java
// 自动类型转换（小→大）
int a = 10;
double b = a;      // int自动转double，安全

// 强制类型转换（大→小）
double c = 3.99;
int d = (int) c;   // d=3，小数直接被截断（不是四舍五入！）

// byte/short/char参与运算自动提升为int
byte x = 10, y = 20;
int result = x + y;   // 必须用int接收
```

**为什么byte运算要提升为int？**
Java虚拟机没有专门处理byte运算的指令，最小运算单位就是int。这是JVM设计决定的，防止byte溢出。

### 字符串拼接

```java
"Hello" + "World"  → "HelloWorld"
"年龄:" + 18       → "年龄:18"
1 + 2 + "3"        → "33"   // 从左到右：先算1+2=3，再拼"3"
"1" + 2 + 3        → "123"  // 从左到右：先拼"1"+2="12"，再拼"12"+3
```

**为什么 + 既能做加法又能拼字符串？**
运算符重载。Java设计者认为用同一个符号更直观（其他语言如PHP用 `.` 拼接反而容易混）。

### 逻辑运算符：为什么有短路逻辑？

```java
// && 短路与：左边false → 右边不执行
if (obj != null && obj.getValue() > 10) {
    // 如果obj是null，后半段根本不会执行，避免了NullPointerException！
}

// & 不短路：两边都一定执行
if (obj != null & obj.getValue() > 10) {
    // 即使obj是null，右边也会执行 → 空指针异常！
}
```

**日常开发几乎永远用 `&&` 和 `||`**，`&` 和 `|` 主要用于位运算。

### 三元运算符

```java
int max = a > b ? a : b;   // 等价于 if-else，但更简洁
```

**为什么要三元运算符？**
有些场景（如变量赋值、return语句），if-else写起来啰嗦，一行三元更简洁。但不能滥用，嵌套多层三元运算符反而更难读。

## 2.7 引用数据类型

### 为什么要有引用类型？

```java
// 基本类型：存值本身
int a = 10;
// 4字节直接存10

// 引用类型：存的是地址
int[] arr = new int[1000000];
// 栈中的arr只存4字节地址 → 1000000个int在堆里
```

**三个核心原因**：

1. **大小不固定**：数组、字符串的长度不确定，没法预先分配固定的栈空间，只能放堆里
2. **节省栈空间**：栈空间很小（通常1~2MB），大数据必须放堆里
3. **共享数据**：多个变量指向同一个对象，一处修改处处可见

```java
// 共享数据的实际用途
ArrayList<User> list = new ArrayList<>();
User u = new User("张三");
list.add(u);    // list里存的是地址，不是复制一份User
u.setName("李四");
// list里的也变成李四了！数据是共享的
```

**你之前问的：为什么不用数组用集合？**

```java
// 数组：长度固定，创建后不能变
int[] arr = new int[5];   // 只能存5个，第6个存不进去
arr[0] = 10; arr[1] = 20; // 需要手动管理索引

// ArrayList：长度自动增长
ArrayList<Integer> list = new ArrayList<>();
list.add(10);   // 不需要管索引
list.add(20);   // 随便加，不够了自动扩容
list.add(30);
// 动态变化的数据（比如用户注册），必须用集合！
```
