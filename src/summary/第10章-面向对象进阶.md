# 第10章：面向对象进阶（static / final / enum）

---

## 10.1 static 静态变量

### 为什么需要 static？

**没有static时**：
```java
public class Student {
    String name;
    String school;  // 每个学生的学校都一样，但每个对象都得存一份
}

Student s1 = new Student();
s1.school = "黑马大学";
Student s2 = new Student();
s2.school = "黑马大学";   // 同样数据存了2份，浪费内存！
// 100个学生就要存100遍"黑马大学"
```

**有了static后**：
```java
public class Student {
    String name;                // 实例变量（每人不同）
    static String school;       // 静态变量（所有人共享，只存一份）
}

Student.school = "黑马大学";    // 类名直接设，所有对象共享
Student s1 = new Student();
Student s2 = new Student();
System.out.println(s1.school);  // 黑马大学
System.out.println(s2.school);  // 黑马大学（同一份数据）
// 100个学生也只存1遍"黑马大学"
```

**static 的本质**：
- 属于类，不属于对象。类加载时就在方法区分配好了，比任何对象都先存在
- 所有对象共享同一份数据
- 用"类名.变量名"直接访问，不需要创建对象

### 什么时候用 static？

```java
// ✅ 适合用static：所有实例都一样的数据
static String nation = "中国";        // 所有人的国籍都一样
static final int MAX_HP = 100;       // 常量

// ❌ 不适合用static：每个实例都不一样的数据
String name;   // 每个人名字不同 → 实例变量
int age;       // 每个人年龄不同 → 实例变量
```

## 10.2 static 修饰方法（工具类）

### 为什么要有工具类？

```java
// 有些方法跟任何具体对象都没关系，是属于"通用功能"的
// 比如：求最大值、判断质数、数组排序...

// 不用static：每次都得创建对象，浪费
MathUtils util = new MathUtils();
int max = util.max(10, 20);   // 还得new，麻烦

// 用static：直接用类名调用
int max = MathUtils.max(10, 20);  // 干净利落
```

**工具类标准写法**：
```java
public class MathUtils {
    // ① 私有化构造方法 → 防止别人new（这个类不需要对象）
    private MathUtils() {}

    // ② 所有方法都是 public static
    public static int max(int a, int b) {
        return a > b ? a : b;
    }

    public static boolean isPrime(int n) {
        if (n <= 1) return false;
        for (int i = 2; i <= Math.sqrt(n); i++) {
            if (n % i == 0) return false;
        }
        return true;
    }
}
```

### static 方法的限制

```java
public class Demo {
    int num = 10;               // 实例变量
    static int count = 100;     // 静态变量

    // 静态方法
    public static void staticMethod() {
        System.out.println(count);     // ✅ 可以访问静态变量
        // System.out.println(num);    // ❌ 不能访问实例变量
        // System.out.println(this);   // ❌ 不能用this
    }

    // 为什么？因为静态方法属于类，通过"类名.方法()"调用时
    // 可能还没有任何对象存在，num 和 this 从哪来？
}
```

**一句话总结**：
- 静态方法里不能访问非静态的东西（因为静态方法可能在没有对象时就被调用）
- 非静态方法里可以访问静态的东西（因为类早就加载好了）

## 10.3 final 关键字

### 为什么需要 final？

```java
// 有些东西你不想让别人改
// 比如说：圆周率π，一天24小时，一周7天...

// 不用final：可能被误改
double pi = 3.14;    // 别人可能在某个角落给你改成 3.0
pi = 3.0;            // 编译通过，但逻辑错了！

// 用final：编译就拦住了
final double PI = 3.14159;
PI = 3.0;            // ❌ 编译报错！常量不能改
```

**final 三种用法**：
```java
// ① final变量 → 常量，只能赋值一次
final int MAX = 100;
final double PI = 3.14;

// ② final方法 → 子类不能重写
public class Parent {
    public final void importantMethod() {
        // 这个方法的逻辑不允许子类改
    }
}

// ③ final类 → 不能被继承
public final class String {   // JDK的String就是final的
    // String的核心逻辑不允许任何人改
}
```

**为什么要用final？**
- 保护核心逻辑不被篡改（安全）
- 编译器可以优化final变量的性能
- 代码更清晰：一眼就知道这是个常量

## 10.4 枚举 enum

### 为什么需要枚举？

**没有枚举时（用常量代替）**：
```java
// 用int常量表示方向
public static final int NORTH = 1;
public static final int SOUTH = 2;
public static final int EAST = 3;
public static final int WEST = 4;

// 问题1：可以传任意数字，没有约束
move(999);   // 编译通过，但999是什么方向？没法控制

// 问题2：常量和数据分离，不好管理
```

**有了枚举后**：
```java
public enum Direction {
    NORTH, SOUTH, EAST, WEST;   // 就这4个值，多了没有
}

// 使用时：类型安全，只能传枚举里的值
public void move(Direction d) { }

move(Direction.NORTH);   // ✅
move(999);               // ❌ 编译报错！类型不匹配
```

### 带属性的枚举

```java
// 游戏中的敌人类型，每个类型有自己的基础属性
public enum EnemyType {
    GOBLIN(80, 20, "哥布林"),       // 血量、攻击力、显示名
    SKELETON(100, 30, "骷髅兵"),
    DRAGON(200, 50, "恶龙"),
    BOSS(300, 80, "魔王");

    // 每个枚举值都自带这些属性
    private int hp;
    private int attack;
    private String displayName;

    // 枚举的构造方法默认private（不让外部new）
    private EnemyType(int hp, int attack, String displayName) {
        this.hp = hp;
        this.attack = attack;
        this.displayName = displayName;
    }

    public int getHp() { return hp; }
    public int getAttack() { return attack; }
    public String getDisplayName() { return displayName; }
}

// 使用：把数据和类型绑定在一起，不会乱
EnemyType e = EnemyType.DRAGON;
System.out.println(e.getDisplayName());  // "恶龙"
System.out.println(e.getHp());           // 200
System.out.println(e.getAttack());       // 50
```

**为什么用枚举而不用常量类？**

| 方式 | 问题 |
|------|------|
| int常量 | 可以传任意int，不安全 |
| String常量 | 可以传任意String，拼写错误发现不了 |
| 枚举 | 编译器帮你检查，只能传定义好的值 |

**一句话**：枚举把"有限的几个选项"变成类型，编译器帮你保证正确性。
