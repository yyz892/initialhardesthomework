# 第一部分：继承

## 11.1 为什么需要继承？

**没有继承时**：写一个Hero类，再写一个Enemy类，它们有很多重复代码。

```java
// 英雄类
public class Hero {
    private String name;
    private int hp;
    private int attack;
    private int defense;

    public void takeDamage(int damage) {
        int actual = damage - defense;
        hp -= actual;
        System.out.println(name + "受到" + actual + "点伤害");
    }
    // get/set 一大堆...
}

// 敌人类 → 和英雄几乎一模一样！重复写了一遍！
public class Enemy {
    private String name;
    private int hp;
    private int attack;
    private int defense;

    public void takeDamage(int damage) {   // 完全一样的代码
        int actual = damage - defense;
        hp -= actual;
        System.out.println(name + "受到" + actual + "点伤害");
    }
    // get/set 又写一遍...
}
```

**问题**：
- 代码重复：takeDamage、get/set 写了两遍
- 修改麻烦：如果受伤逻辑要改，两个类都得改
- 关系表达不清晰：Hero和Enemy都是"游戏角色"，代码上看不出来

**有了继承后**：把共性的东西提到父类，子类只写自己独特的东西。

```java
// 父类：提取共性
public class GameCharacter {
    private String name;
    private int hp;
    private int attack;
    private int defense;

    public void takeDamage(int damage) {  // 只写一次！
        int actual = damage - defense;
        if (actual < 1) actual = 1;
        hp -= actual;
        System.out.println(name + "受到" + actual + "点伤害");
    }
    // get/set 也只写一次
}

// 子类：只需要写自己独特的东西
public class Hero extends GameCharacter {
    private int score;  // 英雄独有的积分

    public void enhanceAttribute() {
        // 英雄独有的属性增强
    }
}

public class Enemy extends GameCharacter {
    private String type;  // 敌人独有的类型

    public void aiAttack() {
        // 敌人独有的AI行为
    }
}
```

**继承的核心意义**：
- **代码复用**：公共代码写一次，所有子类都能用
- **关系表达**：Hero is a GameCharacter（英雄是一个游戏角色），代码结构反映了真实世界的逻辑
- **易于扩展**：加一个新角色只需要写独有部分，公有部分继承父类的就行

## 11.2 Java继承的特点

1. **单继承**：一个子类只能有一个直接父类（但一个父类可以有多个子类）
2. **多层继承**：A → B → C（C继承B，B继承A，C间接拥有A的成员）
3. **Object是所有类的祖宗**：任何类没写extends，默认继承Object

```java
// 只支持单继承
public class Hero extends GameCharacter { }        // ✅
public class Hero extends GameCharacter, Player { } // ❌ 不能同时继承两个

// 但可以多层继承
public class Animal { }
public class Mammal extends Animal { }
public class Dog extends Mammal { }  // Dog间接拥有Animal的方法

// Object提供的基本方法
Object obj = new Hero();
obj.toString();  // 转字符串
obj.equals(other);  // 判断相等
```

### 子类到底继承了父类的哪些东西？

| 父类成员 | 子类能继承吗 | 说明 |
|----------|------------|------|
| 私有成员变量 | ❌ 不能 | private只有本类能访问 |
| 非私有成员变量 | ✅ 能 | |
| 构造方法 | ❌ 不能 | 但子类构造默认调用super() |
| 非私有成员方法 | ✅ 能 | |
| 私有方法 | ❌ 不能 | |

```java
public class Parent {
    private int secret = 10;    // 私有的
    public int money = 100;     // 公开的

    private void privateMethod() { }    // 私有的
    public void publicMethod() { }      // 公开的
}

public class Child extends Parent {
    public void test() {
        // System.out.println(secret);   // ❌ 不能继承私有变量
        System.out.println(money);       // ✅ 继承了公开变量
        // privateMethod();              // ❌ 不能继承私有方法
        publicMethod();                  // ✅ 继承了公开方法
    }
}
```

## 11.3 方法重写 (Override)

### 为什么需要重写？

父类的方法不能满足子类的需求时，子类重写它。

```java
public class Animal {
    public void makeSound() {
        System.out.println("动物发出声音");  // 默认实现，太笼统
    }
}

// 如果不重写：
Dog dog = new Dog();
dog.makeSound();  // "动物发出声音"  → 不对，狗应该汪汪叫！

// 重写后：
public class Dog extends Animal {
    @Override   // @Override注解：让编译器帮你检查是否真的在重写
    public void makeSound() {
        System.out.println("汪汪汪！");  // 符合狗的实际情况
    }
}

public class Cat extends Animal {
    @Override
    public void makeSound() {
        System.out.println("喵喵喵！");  // 符合猫的实际情况
    }
}
```

**重写的规则**：
1. 方法名 + 参数列表必须完全相同
2. 返回值类型：相同或更小（子类型）
3. 权限：不能比父类更低（public > protected > 默认 > private）

```java
public class Parent {
    public void method() { }
    protected Parent getObject() { return this; }
}

public class Child extends Parent {
    @Override
    // public void method() { }       // ✅ 相同权限或更高都可以
    // private void method() { }      // ❌ 不能降低权限

    @Override
    public Child getObject() { return this; }  // ✅ 返回值类型可以是子类型
}
```

### @Override注解有什么用？

```java
public class Parent {
    public void makeSound() { }
}

public class Child extends Parent {
    @Override
    public void makeSound() { }    // ✅ 确实在重写

    // @Override
    // public void makeSoundd() { } // ❌ 方法名写错了，编译报错！
    // 如果不写@Override，上面这行就偷偷变成了一个新方法，不报错
}
```

**@Override = 安全带**：告诉编译器"我是要重写父类方法"，如果写错了（方法名/参数不对），编译器马上报错。

## 11.4 super 关键字

### 为什么需要 super？

子类重写方法后，有时候还需要调用父类原本的功能。

```java
public class GameCharacter {
    private String name;
    private int hp;

    public GameCharacter(String name, int hp) {
        this.name = name;
        this.hp = hp;
    }

    public void attack() {
        System.out.println("普通攻击");
    }
}

public class Hero extends GameCharacter {
    private int power;

    public Hero(String name, int hp, int power) {
        // this.name = name;  // ❌ name是父类private的，子类不能直接访问
        super(name, hp);       // ✅ 调用父类构造方法来赋值
        this.power = power;
    }

    @Override
    public void attack() {
        super.attack();               // 先执行父类的攻击逻辑
        System.out.println("附加额外伤害：" + power);  // 再加上子类自己的逻辑
    }
}
```

**super的三种用法**：

```java
// ① 调用父类构造 → 必须写在子类构造第一行
super(参数);
super();        // 默认就有，不写也行

// ② 调用父类方法 → 子类重写后还想用父类版本
super.方法名();

// ③ 访问父类成员变量 → 子类有同名变量时
System.out.println(super.name);
```

### 为什么要 super() 必须放第一行？

```java
// Java规定：构造方法是自顶向下执行的
// Object的构造先跑 → 然后是父类的 → 最后才是子类的
// 如果父类没有初始化完，子类就用父类的数据，会出问题

public Hero() {
    super();           // 必须先初始化父类
    this.power = 100;  // 然后才能初始化自己的东西
}
```

| 默认(不写) | ✓ | ✓ | ✗ | ✗ | 包内部使用 |

```java
        return "xxx";
    }
}
```

# 第二部分：多态

## 11.6 多态的概念

### 什么是多态？

**多态 = 一个对象有多种形态**。父类引用指向子类对象。

```java
// 一个对象可以以多种"身份"被看待
Dog dog = new Dog();      // dog 是 Dog 类型，正常
Animal a = new Dog();     // dog 也可以被当做 Animal 来看待！

// 这就是多态的核心：
// 编译时看左边的类型（Animal），运行时看右边的类型（Dog）
```

### 为什么需要多态？

**没有多态时**：
```java
// 游戏里有一个喂动物的方法
public void feedDog(Dog dog) { dog.eat(); }
public void feedCat(Cat cat) { cat.eat(); }
public void feedBird(Bird bird) { bird.eat(); }
// 每加一种动物，就得加一个方法！100种动物写100个方法？
```

**有了多态后**：
```java
// 一个方法搞定所有动物！
public void feed(Animal a) {   // 参数是父类类型
    a.eat();                   // 不管传来什么动物，都能喂
}

// 调用时：
feed(new Dog());    // Dog被当做Animal传进去，运行时调用Dog的eat()
feed(new Cat());    // Cat被当做Animal传进去，运行时调用Cat的eat()
feed(new Bird());   // Bird被当做Animal传进去，运行时调用Bird的eat()
// 以后再添加100种动物，feed方法一行都不用改！
```

**这就是多态最大的价值：方法参数用父类，可以接收所有子类对象，代码扩展性极强。**

---

## 11.7 多态的两种形式

### 形式1：父类作为方法参数

```java
// 一个方法接收所有GameCharacter的子类
public static void printStatus(GameCharacter gc) {
    System.out.println(gc.getName() + " HP:" + gc.getHp());
    gc.printHealthBar();
}

// 调用
Hero hero = new Hero();
Enemy enemy = new Enemy();
printStatus(hero);   // ✅ Hero也传得进去
printStatus(enemy);  // ✅ Enemy也传得进去
```

### 形式2：父类作为返回值

```java
// 工厂方法：根据类型返回不同子类对象，但声明返回父类
public static GameCharacter createCharacter(String type) {
    if (type.equals("hero")) {
        return new Hero("勇者", 150, 30);
    } else if (type.equals("goblin")) {
        return new Enemy(EnemyType.GOBLIN);
    }
    return null;
}
```

## 11.8 多态中方法的调用规则

```java
// 核心口诀：编译看左边，运行看右边

Animal a = new Dog();

// 编译时检查：Animal有没有这个方法？
// 运行时执行：调用Dog里的具体实现

a.eat();      // ✅ Animal有eat() → 运行Dog的eat()
a.makeSound(); // ✅ Animal有makeSound() → 运行Dog的makeSound()
// a.wagTail(); // ❌ Animal没有wagTail() → 编译就不通过！
```

**为什么要这么设计？**
- 编译时检查保证安全：你以Animal的身份看这个变量，只能用Animal定义过的方法
- 运行时多态保证灵活：具体执行哪个版本，看实际是哪个子类对象

## 11.9 类型转换（向下转型）

```java
Animal a = new Dog();

// 想看狗的独有方法怎么办？往下转型
if (a instanceof Dog) {    // ① 先判断是不是Dog
    Dog d = (Dog) a;       // ② 安全转换
    d.wagTail();           // ③ 调用Dog独有方法
}

// instanceof：判断实际类型
System.out.println(a instanceof Dog);    // true
System.out.println(a instanceof Animal); // true（Dog也是Animal）
System.out.println(a instanceof Cat);    // false

// ❌ 危险转换 → 会抛 ClassCastException
Dog d = (Dog) new Cat();  // 编译通过，运行报错！→ 所以必须先instanceof
```

# 第三部分：抽象类

## 11.10 抽象类

### 为什么需要抽象类？

父类中有些方法没法给出"合理的默认实现"。

```java
// GameCharacter 父类中的 attack 方法
// 英雄和敌人的攻击方式完全不同，父类根本不知道怎么实现
public class GameCharacter {
    public int attack(GameCharacter target) {
        // 写什么？英雄是烈焰斩，哥布林是偷袭，龙是龙息...
        return 0;  // 随便返回一个？这不合理！
    }
}

// 如果有人在父类没实现好，子类又忘记重写，就出bug了
```

**用抽象类解决**：

```java
public abstract class GameCharacter {   // abstract → 抽象类，不能new
    private String name;
    private int hp;

    // 抽象方法：没有 {} 方法体，子类必须重写！
    public abstract int attack(GameCharacter target);

    // 具体方法：有方法体，子类继承直接用
    public void takeDamage(int damage) {
        hp -= damage;
    }
}

// 子类必须实现抽象方法，不然编译报错
public class Hero extends GameCharacter {
    @Override
    public int attack(GameCharacter target) {
        return (int)(getAttack() * 1.5);  // 必须实现！
    }
    // 如果不实现attack()，编译就报错 → 强制约束！
}
```

**抽象类 vs 普通父类**：

| | 普通父类 | 抽象类 |
|------|---------|--------|
| 能new吗 | ✅ 能 | ❌ 不能 `new GameCharacter()` 报错 |
| 抽象方法 | 不能有 | ✅ 可以有 |
| 子类约束 | 随意重写（容易忘） | 强制重写（编译器逼你） |
| 使用场景 | 子类行为差异不大 | 子类行为完全不同，父类无法给出默认实现 |

**为什么要用抽象类？**
- 强制约束子类必须实现某些方法（编译期保证，不是运行时发现）
- 抽象类本身不完整，不能实例化也是合理的
- 语义清晰：这个类生来就是给人继承的，不是拿来new的

# 第四部分：接口

## 11.11 接口 interface

### 为什么需要接口？

**场景：游戏的技能系统**

有些能力不是某个类的专属，而是跨多个不相关类的共同能力。

```java
// 飞行能力 → Hero能飞，Dragon能飞，Bird能飞
// 游泳能力 → Hero能游泳，Fish能游泳
// 但Hero属于GameCharacter体系，Dragon属于Enemy体系，Bird属于Animal体系
// 它们没有共同的父类！没法用继承来表达"能飞"

// 接口：定义一种"能力"，谁实现了谁就有这个能力
public interface Flyable {
    void fly();    // 接口中的方法默认是 public abstract
}

public interface Swimable {
    void swim();
}

// 类可以实现多个接口（多实现，解决了单继承的限制）
public class Hero extends GameCharacter implements Flyable, Swimable {
    @Override
    public void fly() {
        System.out.println("英雄使用飞行术！");
    }

    @Override
    public void swim() {
        System.out.println("英雄游泳！");
    }
}

public class Dragon extends Enemy implements Flyable {
    @Override
    public void fly() {
        System.out.println("恶龙展翅高飞！");
    }
}
```

**继承 vs 接口的语义区别**：
- 继承（extends）：**is-a** 关系 — "一个英雄**是一个**游戏角色"
- 接口（implements）：**can-do** 关系 — "一个英雄**能做**飞行这件事"

---

## 11.12 JDK8+ 接口新特性

### 为什么加入默认方法和静态方法？

**问题**：接口发布后如果需要加新方法，所有实现类都得改，不然编译报错。

```java
// JDK8的接口最初只有 fly()
public interface Flyable {
    void fly();
}

// 后来想加一个 floatOnAir() 方法
// 如果直接加抽象方法 → 所有实现类全报错，都要改！
public interface Flyable {
    void fly();
    // void floatOnAir();  // ❌ 所有实现类都得加这个方法
}

// JDK8解决方案：默认方法
public interface Flyable {
    void fly();

    // default：有默认实现，实现类可以不改
    default void floatOnAir() {
        System.out.println("悬浮在空中");
    }
}
```

**JDK8+ 接口完整写法**：

```java
public interface Swimable {
    // ① 抽象方法（默认 public abstract，可以不写）
    void swim();

    // ② 默认方法（实现类可选重写）
    default void dive() {
        System.out.println("潜水");
    }

    // ③ 静态方法（接口名直接调用）
    static void info() {
        System.out.println("这是游泳接口 v2.0");
    }

    // ④ 常量（默认 public static final）
    int MAX_DEPTH = 1000;
}

// 使用
Swimable.info();  // 接口名直接调用静态方法
```

## 11.13 抽象类 vs 接口（对比总结）

| 维度 | 抽象类 | 接口 |
|------|--------|------|
| 语义 | **是什么**（is-a） | **能做什么**（can-do） |
| 继承数量 | 单继承 | 多实现 |
| 构造方法 | ✅ 有 | ❌ 没有 |
| 成员变量 | 任意类型 | 只能 public static final 常量 |
| 方法 | 抽象 + 具体都可以 | 抽象 + default + static |
| 什么时候用 | 子类有大量共性代码要共享 | 定义跨层次的"能力"规范 |

**实际选择技巧**：
- 如果子类之间有很多共享代码 → 抽象类
- 如果只定义规范/能力，不管具体代码 → 接口
- 不确定的时候 → 先选接口（更灵活，不占唯一的继承名额）

# 第五部分：内部类

## 11.14 什么是内部类？为什么需要它？

**内部类 = 定义在另一个类里面的类**。

```java
// 外部类
public class Hero {
    // 内部类
    public class Weapon {
        // ...
    }
}
```

### 为什么要有内部类？

**场景1：一个类只被另一个类使用，不需要单独暴露**

```java
// 没有内部类时：Weapon也得单独写一个文件
public class HeroWeapon {  // 这个类只有Hero用，却暴露在外面
    String name;
    int attackBonus;
}

// 有内部类时：Weapon藏在Hero里面，外面看不到
public class Hero {
    private String name;

    // Weapon只有Hero会用，放在里面更合理
    private class Weapon {
        String weaponName;
        int attackBonus;
    }

    private Weapon weapon = new Weapon();  // Hero内部使用
    // 外部代码根本不知道Weapon的存在
}
```

**场景2：内部类可以直接访问外部类的私有成员**

```java
public class Hero {
    private int hp = 150;      // 外部类的私有变量

    public class Skill {
        public void cast() {
            // 内部类可以直接访问外部类的私有成员！
            System.out.println("当前血量：" + hp);  // ✅ 直接访问！
        }
    }
}

// 为什么可以？因为内部类持有外部类对象的引用（隐式的 Hero.this）
```

---

## 11.15 成员内部类

**定义在类中，方法外**。

```java
public class Outer {
    private int num = 10;

    // 成员内部类
    public class Inner {
        private int num = 20;

        public void show() {
            int num = 30;
            System.out.println(num);             // 30  局部变量
            System.out.println(this.num);        // 20  内部类的成员变量
            System.out.println(Outer.this.num);  // 10  外部类的成员变量
        }
    }
}

// 如何创建成员内部类对象？
Outer outer = new Outer();           // 先创建外部类对象
Outer.Inner inner = outer.new Inner(); // 再通过外部类对象创建内部类对象
```

**关键理解**：成员内部类依赖于外部类对象，没有外部对象就没有内部对象。为什么？因为内部类需要访问外部类的成员变量，得先有外部类对象才行。

---

## 11.16 静态内部类

```java
public class Outer {
    private static int staticNum = 10;
    private int num = 20;

    // 静态内部类
    public static class StaticInner {
        public void show() {
            System.out.println(staticNum);     // ✅ 可以访问外部类的静态成员
            // System.out.println(num);        // ❌ 不能访问非静态成员！
            // 为什么？静态内部类不需要外部对象就能创建
            // 如果连对象都没有，去哪找num？
        }
    }
}

// 创建：不需要外部类对象，直接用外部类名
Outer.StaticInner inner = new Outer.StaticInner();
```

## 11.17 匿名内部类（重要！）

### 为什么需要匿名内部类？

有些类只需要用一次，专门写一个类文件太浪费了。

```java
// 不用匿名内部类：得专门写一个类
public class MyFlyable implements Flyable {
    @Override
    public void fly() {
        System.out.println("飞一下");
    }
}
Flyable f1 = new MyFlyable();  // 专门写一个类的文件，就为了用一次

// 用匿名内部类：当场定义，用完即扔
Flyable f2 = new Flyable() {   // new 接口名() { 实现方法 }
    @Override
    public void fly() {
        System.out.println("飞一下");
    }
};  // ← 注意分号！这是一个语句
f2.fly();  // "飞一下"
```

**匿名内部类的本质**：
- 创建了一个**没有名字的类**，实现了指定的接口或继承了指定的类
- 同时创建了这个类的对象
- 适合只用一次的临时实现

```java
// 更常见的用法：直接作为方法参数
// 不用临时变量，直接传
useFlyable(new Flyable() {
    @Override
    public void fly() {
        System.out.println("匿名对象飞行");
    }
});

// Java中常见的匿名内部类：线程
new Thread(new Runnable() {
    @Override
    public void run() {
        System.out.println("新线程运行");
    }
}).start();
```

## 11.18 内部类总结

| 类型 | 位置 | 能访问外部类成员 | 创建方式 |
|------|------|----------------|---------|
| 成员内部类 | 类中方法外 | ✅ 所有（包括private） | `outer.new Inner()` |
| 静态内部类 | 类中方法外 + static | 只能访问静态成员 | `new Outer.Inner()` |
| 匿名内部类 | 使用时当场创建 | ✅ 方法和变量 | `new 接口/类() { 实现 }` |

**选择指南**：
- 这个类只被一个外部类使用 → 成员内部类
- 不需要访问外部类非静态成员 → 静态内部类（更轻量）
- 只用一次，不想起名字 → 匿名内部类
