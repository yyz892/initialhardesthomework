# 第3章：流程控制

---

## 3.1 为什么需要流程控制？

程序默认从上到下顺序执行。但实际需求不可能只是顺序执行——需要根据条件分叉（if），需要重复做一件事（for/while），需要从多个选项中选择（switch）。流程控制让程序有了"判断"和"循环"的能力。

---

## 3.2 if 语句

```java
// 格式1：单if
if (heart > 100) {
    System.out.println("心跳过快！");
}

// 格式2：if-else
if (score >= 60) {
    System.out.println("及格");
} else {
    System.out.println("不及格");
}

// 格式3：if-else if-else
if (score >= 90) {
    System.out.println("优秀");
} else if (score >= 80) {
    System.out.println("良好");
} else if (score >= 60) {
    System.out.println("及格");
} else {
    System.out.println("不及格");
}
```

### if-else if vs 多个独立if的区别

```java
// ❌ 多个独立if：每个if都会被检查，效率低且可能多个分支同时执行
if (score >= 90) { System.out.println("优秀"); }
if (score >= 80) { System.out.println("良好"); }  // 90分也会进这里！

// ✅ if-else if：只要一个分支匹配，后面都不检查，效率高
if (score >= 90) { System.out.println("优秀"); }
else if (score >= 80) { System.out.println("良好"); }  // 90分不会进来
```

### 为什么条件必须是boolean？

Java里 `if(1)` 这样写会编译报错（不像C语言），这是刻意设计的：**强制显式表达意图，减少bug**。C语言中 `if(x = 5)` 是赋值不是比较，是常见的bug来源。

## 3.3 switch 语句

```java
switch (week) {
    case 1 -> System.out.println("周一");  // JDK14+ 箭头语法
    case 2 -> System.out.println("周二");
    default -> System.out.println("无效");
}
```

### switch vs if-else if：什么时候用哪个？

```java
// ✅ 用 switch：判断的是同一个变量的不同值
switch (choice) {
    case 1 -> register();
    case 2 -> login();
    case 3 -> exit();
}

// ✅ 用 if-else if：判断条件不同，或需要范围判断
if (score >= 90) { }
else if (score >= 60) { }

// switch 底层用跳转表优化，比一连串if-else快
// 且代码更清晰：一眼看出在判断"同一个变量的多个等值情况"
```

### 为什么传统switch有"穿透"问题？

```java
// 传统写法（不加break会穿透）
switch (n) {
    case 1:
    case 2:
        System.out.println("1或2");  // 1和2都会执行到这里
        break;
}
// 这是C语言的遗留设计，Java保持了兼容。
// JDK14的箭头语法彻底解决了这个问题（不穿透）。
```

## 3.4 for 循环

```java
// for循环的本质：三个部分
for (初始化; 条件判断; 步进) {
    // 循环体
}

// 执行过程：
// 初始化(只执行1次) → 判断 → 执行循环体 → 步进 → 判断 → 循环体 → ...
```

### 什么时候用 for，什么时候用 while？

```java
// ✅ for：明确知道循环次数
for (int i = 0; i < 100; i++) { }        // 循环100次
for (int i = 0; i < arr.length; i++) { }  // 遍历数组

// ✅ while：不知道循环次数，取决于条件
while (true) {
    // 猜数字，猜对了break，不知道猜几轮
}
```

---

## 3.5 while 和 do-while

```java
// while：先判断再执行（可能一次都不执行）
while (false) {
    // 永远不会执行
}

// do-while：先执行再判断（至少执行一次）
do {
    // 一定会执行一次
} while (false);
```

### 为什么有do-while？

某些场景"至少要做一次"很合理。比如：先显示菜单，再问用户要不要继续。

---

## 3.6 break 和 continue

```java
// break：终止循环
for (int i = 0; i < 10; i++) {
    if (i == 5) break;  // 循环到此结束
}

// continue：跳过当前这一轮
for (int i = 0; i < 10; i++) {
    if (i == 5) continue;  // 跳过5，继续6,7,8,9
}
```

**为什么要区分break和continue？**
- break = "不干了，走人"（跳出整个循环）
- continue = "这轮不干了，继续下轮"（跳过本次）

## 3.7 嵌套循环

```java
// 九九乘法表：外层控制行，内层控制列
for (int i = 1; i <= 9; i++) {          // 行
    for (int j = 1; j <= i; j++) {       // 列
        System.out.print(j + "×" + i + "=" + (i*j) + "\t");
    }
    System.out.println();  // 每行结束换行
}
```

### 嵌套循环的执行顺序

```
外层i=1: 内层j=1
外层i=2: 内层j=1,2
外层i=3: 内层j=1,2,3
...
外层执行1次，内层执行完整1轮
时间复杂度：O(n²)，大数据量时性能很差
```
