# 第5章：方法

---

## 5.1 为什么需要方法？

```java
// 没有方法：一段逻辑要重复写很多次
// 计算1~100的和
int sum1 = 0;
for (int i = 1; i <= 100; i++) { sum1 += i; }

// 又需要计算1~200的和
int sum2 = 0;                          // 同样的逻辑又写一遍
for (int i = 1; i <= 200; i++) { sum2 += i; }

// 再需要计算50~150的和
int sum3 = 0;                          // 又写一遍...
for (int i = 50; i <= 150; i++) { sum3 += i; }
```

**有了方法后**：

```java
// 把逻辑封装成方法，想算哪段算哪段
public static int sum(int start, int end) {
    int total = 0;
    for (int i = start; i <= end; i++) {
        total += i;
    }
    return total;
}

int r1 = sum(1, 100);    // 一行搞定
int r2 = sum(1, 200);    // 代码复用
int r3 = sum(50, 150);   // 不重复
```

方法的核心价值：**代码复用 + 逻辑封装 + 易于维护**。改一个地方（方法体），所有调用的地方都受益。

## 5.2 方法定义格式

```java
public static int add(int a, int b) {
//  ↑      ↑    ↑   ↑    ↑  ↑
// 权限   静态 返回 方法名 参数 参数
    return a + b;
}
```

### 为什么方法要有返回值类型？

告诉调用者"我会返回什么类型的东西"。如果类型对不上编译器直接报错，防止运行时类型错误。

```java
int result = add(10, 20);   // ✅ int接收int
// String s = add(10, 20);  // ❌ 编译报错，类型不匹配
```

### 为什么需要 void？

有些方法执行操作但不返回结果。

```java
public static void printMenu() {
    System.out.println("1.注册  2.登录  3.退出");
    // 只是打印，不需要返回任何东西
}
```

### 为什么参数是"传值"而不是"传变量本身"？

```java
public static void change(int num) {
    num = 100;
}

int a = 10;
change(a);
System.out.println(a);  // 还是10！没变！

// 为什么？为了保护调用者的数据不被意外修改
// Java的设计哲学：方法不应该偷偷修改调用者的变量
```

## 5.3 方法重载 (Overload)

```java
// 同一功能，不同参数类型
public static int add(int a, int b) { return a + b; }
public static double add(double a, double b) { return a + b; }
public static int add(int a, int b, int c) { return a + b + c; }
```

### 为什么需要重载？

让调用者不用记不同的方法名，同一操作的不同版本共用一个名字。

```java
// 不用重载：调用者得记不同名字
addInts(1, 2);
addDoubles(1.5, 2.5);
addThreeInts(1, 2, 3);

// 用重载：统一叫add，编译器自动匹配正确的版本
add(1, 2);        // 自动调用 add(int, int)
add(1.5, 2.5);    // 自动调用 add(double, double)
add(1, 2, 3);     // 自动调用 add(int, int, int)
```

### 为什么重载跟返回值类型无关？

```java
// 假设重载可以只看返回值（Java不允许！）
public static int getValue() { return 1; }
public static double getValue() { return 1.0; }

getValue();   // 编译器懵了：你要调哪个？
// Java规定：只看方法名+参数列表，不看返回值，因为调用时返回值可能被忽略
```

## 5.4 参数传递深度理解

```java
// 基本类型：传值（复制一份，互不影响）
int a = 10;
change(a);  // a还是10

// 引用类型：传地址（指向同一对象，会互相影响）
int[] arr = {10};
changeArr(arr);     // arr[0]变成了100
// 为什么？changeArr通过地址直接修改了堆中的同一份数据
```

**为什么要区分基本类型和引用类型的传递方式？**
- 基本类型占空间小，直接复制值效率高
- 引用类型可能很大（大数组），复制整个对象太浪费，传地址效率高
- 但要注意：传地址意味着方法能改原始数据（双刃剑）
