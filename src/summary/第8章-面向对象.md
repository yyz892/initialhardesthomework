# 第8章：面向对象编程

---

## 8.1 为什么要有面向对象？

**面向过程（C语言风格）**：
```java
// 把数据和方法分开，互相独立
String name = "张三";
int age = 18;
double score = 95;

printInfo(name, age, score);     // 数据和操作是分离的
addScore(score, 10);
```

**面向对象（Java风格）**：
```java
// 把数据和方法绑定在一起，成为一个整体
Student s = new Student("张三", 18, 95);
s.printInfo();    // 数据就在对象内部，调用自己身上的方法
s.addScore(10);
```

**为什么要面向对象？**
- 现实世界就是由"对象"组成的（人、车、手机都是一个一个的对象）
- 数据和对数据的操作绑在一起，逻辑更清晰，更容易理解和维护
- 代码复用：写好一个类，可以反复创建对象使用

---

## 8.2 类和对象的关系

```
类 = 设计图（描述一类事物有哪些属性和行为）
对象 = 根据设计图造出来的实物（具体的、可用的）

比如：
  类：汽车设计图（有颜色、品牌、速度...有启动、刹车...）
  对象：你买的那辆红色宝马（颜色=红、品牌=宝马、速度=0）
```

```java
// 类：模板
public class Student {
    String name;      // 属性：叫什么名字
    int age;          // 属性：多大岁数

    public void study() {  // 行为：能学习
        System.out.println(name + "在学习");
    }
}

// 对象：具体实例
Student s1 = new Student();  // 造一个学生出来（默认值：name=null, age=0）
s1.name = "张三";
s1.age = 18;
s1.study();   // 这个学生在学习

Student s2 = new Student();  // 再造一个学生
s2.name = "李四";
s2.age = 20;     // 两个学生各管各的，互不影响
```

**为什么要用new创建对象？**
- new 在堆内存中为对象分配空间，返回这个空间的地址
- 没有new就没有实际的对象，类只是模板

## 8.3 封装 — private 关键字

### 为什么需要封装？

**没有封装时**：
```java
public class Student {
    String name;
    int age;     // 外部可以直接改，想改多少改多少
}

// 别人可以这样写
Student s = new Student();
s.age = -100;    // ❌ 年龄可以是负数，明显不合理！
s.age = 9999;    // ❌ 也没法阻止
```

**有了封装后**：
```java
public class Student {
    private String name;   // private：外部不能直接访问
    private int age;

    // 通过set方法控制赋值逻辑
    public void setAge(int age) {
        if (age >= 0 && age <= 150) {
            this.age = age;
        } else {
            System.out.println("年龄不合法！必须0~150之间");
        }
    }

    public int getAge() {
        return age;
    }
}

// 使用时
Student s = new Student();
s.age = -100;    // ❌ 编译报错！外部不能直接访问private
s.setAge(-100);  // 走set方法，被拦截："年龄不合法！"
```

**封装的核心思想**：
- 把属性藏起来（private），只暴露安全的方法（get/set）
- 可以在set方法里加校验，防止非法数据
- 外部使用者不需要知道内部细节，只需要知道怎么调用

**打个比方**：你的手机有复杂的内部电路（private），但你不需要知道电路怎么工作的，你只需要用屏幕上的按钮（public方法）操作就行。

## 8.4 this 关键字

### 为什么需要 this？

```java
public class Student {
    private String name;

    // 不用this：参数必须改名，别扭
    public void setName(String n) {    // 参数叫n，不直观
        name = n;
    }

    // 用this：参数名可以和成员变量名一样，更直观
    public void setName(String name) { // 参数叫name，语义清晰
        this.name = name;
        // ↑        ↑
        // 成员变量   参数（局部变量）
    }
}
```

**this的本质**：
```java
// this 就是"当前调用这个方法的对象"的地址
Student s1 = new Student();
s1.setName("张三");   // 这次调用，this = s1

Student s2 = new Student();
s2.setName("李四");   // 这次调用，this = s2

// this 帮你区分"我自己的name"和"传进来的name"
```

**什么时候必须用this？**
- 局部变量和成员变量同名时，必须用this区分
- 构造方法中调用另一个构造方法：`this(参数)`

## 8.5 构造方法

### 为什么需要构造方法？

**没有构造方法时**：
```java
Student s = new Student();
s.setName("张三");   // 每次都要调用set，麻烦
s.setAge(18);
s.setScore(95);
// 创建一个对象要写4行
```

**有了有参构造方法后**：
```java
Student s = new Student("张三", 18, 95);
// 一行搞定！创建对象的同时就赋好初值
```

```java
public class Student {
    private String name;
    private int age;

    // 无参构造：没写的时候Java默认给一个
    // 但如果你写了有参构造，无参构造就没了，必须手动补！
    public Student() {
    }

    // 有参构造：创建对象时直接赋值
    public Student(String name, int age) {
        this.name = name;
        this.age = age;
    }
}

// 两种创建方式
Student s1 = new Student();                // 用无参
Student s2 = new Student("张三", 18);      // 用有参，直接赋值
```

**构造方法的特点**：
- 方法名必须和类名一模一样
- 没有返回值（连void都不写）
- new 对象时自动调用，且只调用一次
- 支持重载（可以写多个有参构造）

### 为什么写了有参构造要补无参构造？

```java
// 很多框架（如Spring、MyBatis）默认用无参构造创建对象
// 如果你的类没有无参构造，框架就报错了
// 养成习惯：有参+无参都写上 → JavaBean标准
```

## 8.6 JavaBean标准规范

```java
// JavaBean = 标准的数据类
// 规范：
// ① public类
// ② private成员变量
// ③ 必须有无参构造
// ④ 每个成员变量提供 get/set 方法

public class User {
    private String username;
    private String password;

    public User() {}   // 无参 → 必须有

    public User(String username, String password) {  // 有参 → 方便创建
        this.username = username;
        this.password = password;
    }

    // get/set → 封装
    public String getUsername() { return username; }
    public void setUsername(String username) { this.username = username; }
    public String getPassword() { return password; }
    public void setPassword(String password) { this.password = password; }
}
```

**为什么要遵循JavaBean规范？**
- 所有Java开发者都认这个规范，协作效率高
- 框架（Spring/MyBatis）默认按这个规范反射调用
- IDEA可以一键生成 get/set/构造方法（Alt+Insert）

## 8.7 为什么用引用数据类型？

### 基本类型 vs 引用类型的内存区别

```java
// 基本类型：变量直接存值
int a = 10;      // 栈中：a → 10
int b = a;       // 栈中：b → 10 （复制了一份值）
b = 20;          // a还是10，b变成20，互不影响

// 引用类型：变量存的是地址
int[] arr1 = new int[]{1, 2, 3};
// 栈中：arr1 → 0x001（地址）
// 堆中：[1, 2, 3] 在0x001位置

int[] arr2 = arr1;
// 栈中：arr2 → 0x001（和arr1指向同一个地址！）
arr2[0] = 100;
// arr1[0] 也变成100了！因为arr1和arr2指向同一块堆内存
```

**为什么要有引用类型？**

1. **节省内存**：基本类型存值本身（小），引用类型只存4/8字节的地址，实际数据在堆里
2. **动态大小**：数组、字符串、对象这些大小不固定的数据只能放堆里，用引用指向它们
3. **共享数据**：多个变量可以指向同一个对象，一处修改处处可见
4. **复杂结构**：对象嵌套对象（如 User 里有 Address），必须用引用才能实现

```java
// 实际例子：为什么用引用
User u = new User("张三", "123456");
// u只是存了一个4字节的地址
// 真正的User对象（name、password等）在堆里
// 如果User是基本类型，那它得占多大？不现实

// 共享数据的例子
ArrayList<User> list = new ArrayList<>();
list.add(u);     // list里存的是u的地址，不是复制一份User
u.setPassword("999999");
// list里这个User的密码也变成999999了（同一份数据）
```
