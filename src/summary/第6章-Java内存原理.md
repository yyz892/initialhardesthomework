# 第6章：Java内存原理

---

## 6.1 为什么需要理解内存？

```java
int[] arr1 = {1, 2, 3};
int[] arr2 = arr1;
arr2[0] = 100;
System.out.println(arr1[0]);  // 100！
// 如果不理解内存，这个结果会让人困惑
```

理解内存才能写出正确的代码，排查Bug时有据可依。

---

## 6.2 内存三大区域

```
┌────────────┐
│   栈 Stack  │ 方法执行时用
│            │ 存：局部变量（基本类型值、引用地址）
│            │ 特点：方法结束自动销毁，空间小(1~2MB)
├────────────┤
│   堆 Heap   │ 存储对象和数组
│            │ 存：所有 new 出来的东西
│            │ 特点：GC垃圾回收管理，空间大
├────────────┤
│ 方法区      │ 存储类信息
│ Method Area │ 存：.class字节码、static变量
│            │ 特点：类加载时分配，JVM销毁时释放
└────────────┘
```

### 为什么分栈和堆？

- **栈**：小而快，先进后出，方法调用天然适合栈结构
- **堆**：大而灵活，存放大小不确定的数据，由GC管理生命周期
- 分开设计是为了各司其职，高效利用内存

## 6.3 基本类型内存

```java
public static void main(String[] args) {
    int a = 10;
    int b = a;     // 把10复制一份给b
    b = 20;        // 改b不影响a
    // 结论：基本类型 = 独立的副本，互不干扰
}
```

```
栈-内存图：
main方法栈帧中：
  a → 10
  b → 20  (自己单独的空间)
```

## 6.4 数组内存（理解引用类型的关键）

```java
int[] arr1 = new int[]{1, 2, 3};
int[] arr2 = arr1;       // 把地址复制给arr2（不是复制数组！）
arr2[0] = 100;
System.out.println(arr1[0]);  // 100
```

```
栈：                堆：
arr1 → 0x100       0x100: [100, 2, 3]  ← arr1和arr2指向同一块！
arr2 → 0x100

操作arr2[0]本质上是操作堆中0x100位置的数组
arr1也指向同一块，所以arr1[0]也变了
```

### 对比：基本类型 vs 引用类型

```java
// 基本类型：独立空间
int a = 10;  int b = a;  b = 20;   // a=10不变

// 引用类型：共享空间
int[] x = {1};  int[] y = x;  y[0] = 2;  // x[0]=2也变了
```

## 6.5 对象内存

```java
User u = new User("张三", 18);
```

```
步骤：
1. 加载 User.class 到方法区
2. main方法进栈
3. new → 堆中开辟空间
4. 成员变量赋默认值 (name=null, age=0)
5. 调用构造方法 User(name, age) 赋值
6. 返回堆地址存到栈中的u变量
```

### 为什么对象在堆里？

- 对象大小不确定（成员变量可能很多）
- 生命周期不确定（创建后可能被多个方法使用）
- 栈太小存不下大对象

---

## 6.6 方法调用内存

```java
public static void main(String[] args) {
    int result = add(1, 2);     // add进栈
    System.out.println(result);  // add已出栈
}

public static int add(int a, int b) {
    return a + b;
}
```

```
调用过程：
main栈帧(在栈底) → 调用add → add栈帧(压到栈顶) → add执行完 → add栈帧弹出销毁
```

### 为什么方法执行完局部变量就没了？

方法出栈时整个栈帧被销毁，栈帧里的局部变量自然消失。这是自动的、不可逆的。这就是"局部变量作用域"的底层原理。
