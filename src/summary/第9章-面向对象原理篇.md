## 9.1 new一个对象到底发生了什么？

```java
Student s = new Student("张三", 18);
```

```
详细步骤：
┌──────────────────────────────────────┐
│ 1. 方法区：加载 Student.class 字节码 │
├──────────────────────────────────────┤
│ 2. 栈：main方法进栈                  │
├──────────────────────────────────────┤
│ 3. 堆：new → 开辟内存空间            │
│    成员变量自动赋默认值：             │
│      name = null                     │
│      age = 0                         │
├──────────────────────────────────────┤
│ 4. 构造方法入栈执行：                │
│      Student(String name, int age)   │
│      把默认值改成传入的值             │
│      name = "张三"                   │
│      age = 18                        │
│    构造方法出栈                       │
├──────────────────────────────────────┤
│ 5. 堆地址(0x100)返回给栈中的变量s    │
│    栈：s → 0x100                     │
└──────────────────────────────────────┘
```

## 9.2 多个对象的内存关系

```java
Student s1 = new Student("张三", 18);
Student s2 = new Student("李四", 20);
```

```
栈：                  堆：
s1 → 0x100       0x100: {name="张三", age=18}
s2 → 0x200       0x200: {name="李四", age=20}

两个对象在堆中独立存在，互不影响
但同一个类的字节码只有一份（方法区共享）
```

### 为什么不同对象共用同一份字节码？

```java
// 100个Student对象，只有1份Student.class在方法区
// 每个对象只存自己的数据（name, age），方法代码共享
// 如果每个对象都存一份方法代码，内存早就爆了
```

## 9.3 对象作为方法参数

```java
public static void changeAge(Student s) {
    s.setAge(100);  // 通过地址直接修改堆中对象的age
}

Student s = new Student("张三", 18);
changeAge(s);
System.out.println(s.getAge());  // 100，被改了！
```

```
详细过程：
1. main方法栈帧：s → 0x100 (堆地址)
2. 调用changeAge(s) → 把地址0x100传给形参s
3. changeAge栈帧：s → 0x100 (和main中的s指向同一个堆对象)
4. s.setAge(100) → 通过地址0x100找到堆中的对象 → 修改age=100
5. changeAge结束出栈，但堆中对象的age已经变了
6. main中的s仍然指向0x100，所以看到的age是100
```

### 为什么引用传递有风险？

```java
// 数组作为参数
public static void clear(int[] arr) {
    for (int i = 0; i < arr.length; i++) {
        arr[i] = 0;  // 原数组被清空了！
    }
}
int[] data = {1, 2, 3, 4, 5};
clear(data);   // data变成{0,0,0,0,0}了！
// 如果不理解引用传递，这种bug很难排查
```

## 9.4 this关键字的本质

```java
public class Student {
    private String name;

    public void setName(String name) {
        this.name = name;
    }
}

// 当调用 s.setName("张三") 时：
// JVM把s的地址赋给this
// this = s的地址(0x100)
// this.name就是s这个对象在堆中的name属性
```

```
实际每个成员方法第一个参数隐式是 this：
JVM内部编码：setName(Student this, String name) {
    this.name = name;
}
调用时：setName(s, "张三")
```
